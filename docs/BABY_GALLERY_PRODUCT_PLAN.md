# 아기 갤러리 제품 기획서

## 1. 제품 요약

아기 갤러리는 어린 아기 또는 유아가 Android 태블릿에서 로컬 사진과 영상을 안전하게 볼 수 있도록 만든 단순한 갤러리 앱이다.

부모는 기기에 사진과 영상을 미리 옮겨두고, 이 앱을 실행한 뒤 Android의 App Pinning 기능으로 앱을 고정한다. 이후 태블릿을 아이에게 건네면 아이는 사진과 영상만 탐색하고 감상할 수 있다.

이 앱은 보기 전용이다. 사진/영상 조회와 재생만 제공하며, 편집, 삭제, 공유, 클라우드 동기화, 계정 로그인, 외부 앱 열기 기능은 제공하지 않는다.

## 2. 핵심 목표

삼성 갤러리 앱과 비슷하게 익숙한 형태의 로컬 미디어 뷰어를 만들되, 기능은 아래 세 가지로 제한한다.

1. 로컬 미디어 조회
2. 로컬 미디어 목록 표시
3. 사진 또는 영상 전체화면 표시

다른 앱으로 나가지 못하게 하는 역할은 기본적으로 Android App Pinning에 맡긴다. 앱은 부모가 App Pinning을 시작하기 쉬운 흐름을 제공하지만, 앱 자체가 OS 수준의 잠금 기능을 대체하려고 하지는 않는다.

## 3. 대상 사용자

### 부모

- 아이가 태블릿으로 가족 사진과 영상을 보게 하고 싶다.
- 사진과 영상은 이미 기기에 복사해둔 상태다.
- 아이가 실수로 사진을 삭제하거나 편집하거나 공유하지 않기를 원한다.
- 아이가 갤러리 외의 앱을 만지지 않기를 원한다.
- Android App Pinning 기능을 사용할 의향이 있다.

### 아이

- 터치 조작이 예측 불가능하다.
- 썸네일을 누르고, 화면을 넘기고, 영상을 보는 정도의 단순한 사용만 한다.
- 복잡한 메뉴나 위험한 액션을 볼 필요가 없다.

## 4. 제품 원칙

- 보기 전용으로 설계한다.
- 아이가 만지는 화면에는 삭제, 편집, 공유 같은 위험한 기능을 두지 않는다.
- 사진/영상 파일을 수정하거나 이동하지 않는다.
- 첫 화면은 바로 사용할 수 있는 갤러리 화면이어야 한다.
- 랜딩 페이지나 설명용 시작 화면은 만들지 않는다.
- 갤러리 사용 경험은 삼성 갤러리와 유사하게 유지한다.
- 앱 밖으로 나가지 못하게 하는 기능은 Android App Pinning을 적극 활용한다.
- 앱 내부에서는 아이가 실수로 눌러도 돌이킬 수 없는 결과가 생기지 않아야 한다.

## 5. MVP 범위

### 포함할 기능

- 로컬 사진/영상 접근 권한 요청
- Android MediaStore 기반 로컬 이미지/영상 조회
- 전체 미디어 보기
- 앨범 또는 폴더별 보기
- 사진/영상 썸네일 그리드
- 영상 썸네일의 재생 표시 및 길이 표시
- 사진 전체화면 보기
- 영상 전체화면 재생
- 전체화면에서 좌우 스와이프로 이전/다음 미디어 이동
- 전체화면에서 뒤로 가기로 그리드 복귀
- 부모용 "시청 시작" 진입점

- 마지막으로 보던 앨범 또는 화면 복원
- 권한 없음, 미디어 없음, 로딩 상태 처리

- 편집

- 즐겨찾기
- 휴지통
- 파일 이동

- 클라우드 백업 또는 동기화

- 댓글, 태그, 인물, 지도, 추억, 스토리 기능

## 6. 권장 기술 방향

권장 구현 방식은 네이티브 Android 앱이다.

- 플랫폼: Android
- 언어: Kotlin
- UI: Jetpack Compose
- 미디어 조회: MediaStore
- 영상 재생: AndroidX Media3 ExoPlayer
- 이미지/썸네일 로딩: Coil 또는 Compose 친화적인 이미지 로더
- 간단한 설정 저장: DataStore
- 최소 지원 버전: 실제 대상 태블릿에 맞춰 결정하되, 특별한 이유가 없다면 Android 8.0 이상을 현실적인 기준으로 잡는다.

공식 참고 문서:

- Android App Pinning: https://support.google.com/android/answer/9455138
- `Activity.startLockTask()`: https://developer.android.com/reference/android/app/Activity#startLockTask()
- MediaStore 공유 미디어 접근: https://developer.android.com/training/data-storage/shared/media
- Android 14 선택 사진/영상 접근: https://developer.android.com/about/versions/14/changes/partial-photo-video-access
- Media3 ExoPlayer: https://developer.android.com/media/media3/exoplayer

## 7. 권한 정책

앱은 로컬 사진과 영상을 읽는 데 필요한 권한만 요청한다.

### Android 13 이상

- `READ_MEDIA_IMAGES`
- `READ_MEDIA_VIDEO`

### Android 14 이상

- 사용자가 일부 사진/영상만 선택해 접근 권한을 줄 수 있는 흐름을 지원한다.
- 전체 라이브러리 접근이 항상 가능하다고 가정하지 않는다.
- 일부 미디어만 허용된 상태에서도 앱이 정상 동작해야 한다.

### Android 12 이하

- `READ_EXTERNAL_STORAGE`

앱은 편집이나 삭제를 하지 않으므로 쓰기 권한을 요청하지 않는다. 특별한 요구가 생기기 전까지 광범위한 파일 관리 권한도 요청하지 않는다.

### 주요 화면

1. 권한 요청 화면
2. 라이브러리 화면
3. 앨범 미디어 그리드 화면
4. 전체화면 뷰어
5. 부모 설정 화면

부모 설정 화면은 MVP에서는 최소화한다. 아이가 우연히 접근하기 어렵게 숨기거나, 초기 버전에서는 아예 제공하지 않아도 된다.

## 9. 화면 기획

### 9.1 권한 요청 화면

목적:

- 앱이 사진과 영상을 표시하려면 권한이 필요하다는 점을 부모에게 안내한다.

구성:

- 짧은 안내 문구
- 권한 요청 버튼
- 권한 거부 시 설정으로 이동하는 안내

주의:

- 아이가 보는 화면이 아니라 부모가 초기 설정할 때 보는 화면이다.
- 설명은 길지 않게 유지한다.

### 9.2 라이브러리 화면

목적:

- 전체 미디어 또는 앨범/폴더를 선택할 수 있게 한다.

구성:

- "전체 사진 및 영상" 항목
- 앨범/폴더 카드 그리드
- 각 카드에는 대표 썸네일, 앨범명, 미디어 개수를 표시한다.
- "시청 시작" 버튼을 제공한다.

허용되는 동작:

- 전체 미디어 열기
- 특정 앨범 열기
- 시청 시작

허용하지 않는 동작:

- 길게 눌러 선택
- 삭제
- 편집
- 공유
- 외부 앱으로 열기

### 9.3 앨범 미디어 그리드 화면

목적:

- 삼성 갤러리와 유사하게 사진과 영상을 썸네일 그리드로 보여준다.

구성:

- 상단에 앨범명 또는 현재 보기 이름
- 반응형 썸네일 그리드
- 영상 항목에는 재생 아이콘과 영상 길이 표시
- 구현 비용이 크지 않다면 날짜별 묶음 표시를 추가할 수 있다.

허용되는 동작:

- 스크롤
- 썸네일 탭으로 전체화면 열기
- 뒤로 가기로 라이브러리 화면 복귀

허용하지 않는 동작:

- 다중 선택
- 컨텍스트 메뉴
- 드래그 앤 드롭
- 삭제
- 편집
- 공유

### 9.4 전체화면 뷰어

목적:

- 아이가 사진이나 영상을 방해 없이 볼 수 있게 한다.

구성:

- 어두운 배경
- 사진 또는 영상 전체화면 표시
- 영상의 경우 재생/일시정지 오버레이
- 영상 진행 바는 선택 사항
- 사진 확대/축소는 MVP 이후 기능으로 미룰 수 있다.

허용되는 동작:

- 좌우 스와이프로 이전/다음 미디어 이동
- 영상 탭으로 재생/일시정지
- 뒤로 가기로 그리드 복귀

허용하지 않는 동작:

- 공유 시트 열기
- 편집 화면 열기
- 삭제
- 상세 정보 패널
- 다른 앱으로 열기

## 10. App Pinning 흐름

부모가 Android 설정에서 App Pinning을 켜둔다는 전제를 둔다.

권장 흐름:

1. 부모가 아기 갤러리 앱을 실행한다.
2. 부모가 앨범 또는 전체 미디어를 선택한다.
3. 부모가 "시청 시작"을 누른다.
4. 앱이 그리드 또는 첫 번째 미디어를 전체화면/몰입형 보기로 연다.
5. 가능한 경우 앱이 `startLockTask()`를 호출한다.
6. Android가 화면 고정 확인 UI를 표시한다.
7. 부모가 확인한 뒤 태블릿을 아이에게 건넨다.

App Pinning이 꺼져 있거나 자동 시작이 불가능한 경우:

- 부모에게 짧은 안내를 보여준다.
- 아이가 보는 중에는 반복 안내를 보여주지 않는다.
- 앱은 App Pinning 없이도 일반 갤러리처럼 동작해야 한다.

## 11. 기능 요구사항

### FR-001 미디어 권한 요청

앱은 Android 버전에 맞는 사진/영상 읽기 권한을 요청해야 한다.

완료 기준:

- 최초 실행 시 필요한 권한을 요청한다.
- 권한이 허용되면 미디어가 표시된다.
- 권한이 거부되면 앱이 중단되지 않고 안내 화면을 보여준다.
- Android 14 이상에서 일부 미디어만 허용해도 앱이 정상 동작한다.

### FR-002 로컬 미디어 조회

앱은 MediaStore를 통해 로컬 이미지와 영상을 조회해야 한다.

완료 기준:

- 기기의 공유 미디어 저장소에 있는 사진과 영상이 표시된다.
- 최신순 정렬을 지원한다.
- 미디어 조회는 UI 스레드를 막지 않는다.
- 접근할 수 없거나 사라진 파일은 안전하게 건너뛴다.

### FR-003 앨범 목록

앱은 미디어를 앨범 또는 폴더 단위로 묶어 보여줘야 한다.

완료 기준:

- 앨범 목록에 대표 썸네일, 앨범명, 항목 수가 표시된다.
- "전체 사진 및 영상" 항목이 제공된다.
- 특정 앨범을 열면 해당 앨범의 미디어만 표시된다.

### FR-004 미디어 그리드

앱은 삼성 갤러리와 유사한 썸네일 그리드를 제공해야 한다.

완료 기준:

- 태블릿 화면 크기에 맞춰 그리드 열 수가 자연스럽게 조정된다.
- 영상 항목은 사진 항목과 구분된다.
- 많은 미디어가 있어도 스크롤이 부드럽다.
- 썸네일을 누르면 전체화면 뷰어가 열린다.

### FR-005 사진 전체화면 보기

앱은 사진을 전체화면으로 표시해야 한다.

완료 기준:

- 사진이 왜곡 없이 화면에 맞게 표시된다.
- 배경은 어두운 색으로 표시된다.
- 좌우 스와이프로 인접한 미디어로 이동한다.
- 뒤로 가기를 누르면 기존 그리드 위치로 돌아간다.

### FR-006 영상 전체화면 재생

앱은 영상을 전체화면으로 재생해야 한다.

완료 기준:

- 영상을 열면 재생이 시작된다.
- 탭으로 재생/일시정지를 전환할 수 있다.
- 뷰어를 떠나면 플레이어 리소스가 정리된다.
- 좌우 스와이프로 인접한 미디어로 이동할 수 있다.

### FR-007 보기 전용 안전성

앱은 위험하거나 외부로 이어지는 액션을 제공하지 않아야 한다.

완료 기준:

- 삭제 기능이 없다.
- 편집 기능이 없다.
- 공유 기능이 없다.
- 다른 앱으로 열기 기능이 없다.
- 파일 이동, 복사, 이름 변경 기능이 없다.

### FR-008 App Pinning 진입

앱은 부모가 화면 고정 시청 세션을 시작할 수 있는 흐름을 제공해야 한다.

완료 기준:

- "시청 시작" 액션이 있다.
- 가능한 경우 앱이 화면 고정을 시작하도록 요청한다.
- OS 확인이 필요한 경우에도 흐름이 이해 가능하다.
- App Pinning을 사용할 수 없어도 앱은 일반 갤러리로 동작한다.

### FR-009 상태 복원

앱은 마지막으로 사용한 보기 상태를 복원해야 한다.

완료 기준:

- 앱을 다시 열면 마지막 앨범 또는 전체 미디어 화면을 복원한다.
- 마지막 앨범이 사라졌다면 라이브러리 화면으로 돌아간다.

## 12. 비기능 요구사항

### 성능

- 미디어 조회는 UI 스레드 밖에서 수행한다.
- 썸네일은 지연 로딩한다.
- 화면 밖으로 나간 썸네일 요청은 취소 가능해야 한다.
- 수천 개의 미디어가 있어도 앱이 멈추지 않아야 한다.
- 영상 플레이어 리소스는 화면을 떠날 때 정리한다.

### 안정성

- 앱 실행 중 미디어 파일이 외부에서 삭제되어도 앱이 중단되지 않아야 한다.
- 손상되었거나 지원하지 않는 미디어가 있어도 앱이 중단되지 않아야 한다.
- 권한 거부 상태를 정상적으로 처리해야 한다.

### 개인정보

- MVP에서는 분석 도구를 넣지 않는다.
- 네트워크 접근이 필요하지 않다.
- 미디어를 업로드하지 않는다.
- 계정 시스템을 만들지 않는다.

### 아이 사용성

- 터치 영역을 크게 유지한다.
- 아이가 보는 화면에는 작은 메뉴 버튼을 최소화한다.
- 텍스트 설명은 최소화한다.
- 전체화면 뷰어는 충분한 대비를 유지한다.

## 13. 내비게이션 규칙

- 앱 실행 -> 권한이 없으면 권한 요청 화면
- 앱 실행 -> 권한이 있으면 라이브러리 화면
- 라이브러리 화면 -> 전체 미디어 그리드
- 라이브러리 화면 -> 앨범 미디어 그리드
- 미디어 그리드 -> 전체화면 뷰어
- 전체화면 뷰어 뒤로 가기 -> 미디어 그리드
- 미디어 그리드 뒤로 가기 -> 라이브러리 화면

뒤로 가기나 일반 조작이 시스템 파일 선택기, 공유 시트, 편집 화면, 외부 앱 실행으로 이어지면 안 된다.

## 14. 예외 상황

앱은 아래 상황을 처리해야 한다.

- 기기에 사진/영상이 없음
- 사진만 있음
- 영상만 있음
- 권한 거부됨
- Android 14 이상에서 일부 미디어만 허용됨
- SD 카드 또는 외부 저장소가 제거됨
- 앱 실행 중 미디어 파일이 외부에서 삭제됨
- 지원하지 않는 영상 코덱
- 매우 큰 사진 파일
- 화면 회전
- 앱 프로세스가 종료되었다가 복원됨

## 16. 테스트 계획

### 테스트용 미디어 세트

기기에 아래 미디어를 준비한다.

- 사진 20개 이상
- 영상 10개 이상
- 여러 폴더 또는 앨범
- 매우 큰 사진 1개 이상
- 가능하다면 손상되었거나 지원하지 않는 미디어 1개

### 수동 인수 테스트

- 최초 설치 후 권한 요청 흐름
- 권한 거부 흐름
- Android 14 이상 선택 미디어 권한 흐름
- 앨범 목록 표시
- 전체 미디어 그리드 표시
- 특정 앨범 그리드 표시
- 사진 전체화면 열기 및 좌우 스와이프
- 영상 전체화면 열기, 재생, 일시정지, 좌우 스와이프
- 전체화면에서 뒤로 가기
- 그리드에서 뒤로 가기
- 삭제, 편집, 공유, 다른 앱으로 열기 기능이 없는지 확인
- App Pinning 시작 흐름
- 화면 회전 후 정상 동작
- 외부에서 미디어를 삭제한 뒤 앱 안정성 확인

## 17. 완료 기준

제품은 아래 조건을 만족할 때 완료된 것으로 본다.

- 부모가 Android 태블릿에 앱을 설치할 수 있다.
- 앱이 로컬 사진과 영상을 표시한다.
- 아이가 썸네일을 탐색하고 사진/영상을 전체화면으로 볼 수 있다.
- 앱에 삭제, 편집, 공유, 이동, 이름 변경, 업로드, 다른 앱으로 열기 기능이 없다.
- 부모가 App Pinning 기반 시청 세션을 시작할 수 있다.
- 미디어 없음, 권한 거부, 일부 권한 허용 상태를 처리한다.
- 최소 한 대 이상의 실제 Android 기기 또는 에뮬레이터에서 수동 테스트가 완료되었다.
